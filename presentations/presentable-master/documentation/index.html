Table of Contents

Introduction

Presentable is a small script/plugin/widget/thingy that adds a table of contents to HTML based presentations. You can see an example of it in action in this very presentation.

If you're interested in the presentation framework itself, which is kind of awesome, you should talk to this guy. He made it. It's called Reveal.js and as you may have heard, it's kind of awesome.

How Does Presentable Work?

I'm glad you asked.

Presentable creates a table of contents by parsing the HTML of the presentation. It's mainly looking for header tags, but you can identify other elements too as you'll see.

Reveal.js, html5slides, io-2012-slides, Shower and impress.js are supported out of the box and Presentable can theoretically handle other frameworks, although your mileage may vary.

Presentable should work modern browsers like Firefox, Chrome, Safari, WebKit et al and IE9/10

Using Presentable

Basic

  • By default, a "title" is any HTML element that meets the following requirements:
    • It is an <h1>, <h2>, <h3> or any tag marked with the .presentable-title class
  • The first eligible element in the DOM will be the table of contents title 
    <section>
    <h1>This will be the title in the table of contents</h1>
    <h2>This will not</h2>
</section>
  • Every slide should have a "title" or it will be assigned the title "Untitled Slide" in the table of contents.
  • However, untitled slides can be omitted from the table of contents by using the "hideNoTitle" option.

Reveal.js & Nested Slides

  • Nested section tags will be included as sub-titles in the generated table of contents
  • An exception to the "Untitled Slide" rule from the Basic Usage slide is a stack of slides with a main heading: 
    <!-- The first nested section tag does not have a title, it will be omitted
from the TOC and the main heading will be used instead.-->
<section>
    <h2>table of contents title</h2>
    <section>
        <p>No title</p>
    </section>
</section>

Installation Instructions

Reveal.JS

+ Presentable

CSS & HTML

  1. Place the following in the <head> tag: 
    <link rel="stylesheet" href="path/to/presentable.css">
  2. Modify the first <section> tag to look like this 
    <section>
    <section>
        <h1>Your Presentation Title Here</h1>
        <!-- Depending  on your version of reveal.js you may also
        have a script tag in this section -->
    </section>
    <section>
        <h2>Table of Contents</h2>
        <nav id="presentable-toc" class="revealjs"></nav>
    </section>
</section>
  3. Put the icon code below the .slides <div> 
    </div><!--this is the closing .slides div-->

<aside id="presentable-icon" class="revealjs">
    <a title="Table of Contents" href="#/0/1">
        <img alt="Table of Contents" src="path/to/revealjs.png"/>
    </a>
</aside>

JS & Initialization

  1. 4. Use Reveal.initialize() to extend the presentation framework. 
    Reveal.initialize({
    // Other configs...
    dependencies: [
        // Not all dependencies shown...
        { src: 'lib/js/data-markdown.js', condition: function() { return !!document.querySelector( '[data-markdown]' ); } },
        // presentable.js must come after data-markdown.js
        { src: 'path/to/presentable.js', callback: function() { presentable.toc({framework: "revealjs"}); } }
    ]
});
  2. 5. Initialization is done with a callback function. 
    Reveal.initialize({
    // Other configs...
    dependencies: [
        // Not all dependencies shown...
        { src: 'path/to/presentable.js', callback: function() { presentable.toc({framework: "revealjs"}); } }
    ]
});

Configuration

Basic

Several configurations can be passed to presentable.toc(). The options and their default values are demonstrated below.

presentable.toc({
    // Custom presentation structure. See following slide.
    // "framework" is ignored
    data: {"slides": []},

    // Accepts "revealjs", "html5slides", "io-2012-slides", "shower", or "impressjs".
    // "data" is ignored.
    framework: "",

    // keyCode used for keyboard navigation back to TOC slide.
    keyCode: 84     // 84 = 't' key, false to disable

    // Text displayed in TOC when no title is found. See following slide.
    noTitle: "Untitled Slide"

    // Untitled slides are excluded from TOC
    hideNoTitle: false

    // Reload when Presentable link is clicked. For frameworks without navigation API
    reload: false,

    // HTML IDs, classes, and elements to be considered slide titles
    titles: "h1,h2,h3,.presentable-title",

    // URL hash pattern
    urlHash: "#"
});

"framework" or "data" is required. The rest are optional.

Other Frameworks

A JSON object can be passed to presentable.toc() making it possible to generate a table of contents for otherwise unsupported presentation frameworks.

The JSON object should have the following format:

var customData = {"slides": [
    {"index": "1", "title": "slide 1"},
    {"index": "2", "title": "slide 2", "toc": true},
    {"index": "3", "title": "slide 3", "nested": [
        {"index": "3/0", "title": "slide 3/0"},
        {"index": "3/1", "title": "slide 3/1"}
     ]},
    {"index": "4", "title": "slide 4"},
    ...
]};
presentable.toc({
    data: customData,
    // Other config options
});

"toc": true identifies slide containing #presentable-toc.

Internationalization

Presentable can be used with any language!

The following demonstrates the config options you might want to change based on your language.

presentable.toc({
    // Update with your language of choice
    noTitle: "Untitled Slide"

    // Use keyCode suitable for your language / keyboard
    // http://www.asquare.net/javascript/tests/KeyCode.html
    keyCode: 84    // 84 = t
});

Bonus! Getting a Slide Title

Presentable exposes the slideTitle() function which returns the title of a slide identified by an index which matches the URL hash corresponding to that slide.

  1. Name: presentable.slideTitle
  2. Parameters: Index of slide as a string
  3. Return: Title of a slide
  4. Example: 
    presentable.slideTitle("11");  // returns "Bonus! Getting a Slide Title"
presentable.slideTitle("0/1"); // returns "Table of Contents"

One possible use is to track traffic to specific slides by passing the slide title to your web analytics provider each time it is visited.
